Vimoir is a library that can be used to build Netbeans applications for Vim and that includes an example allowing to hear Vim.

Use the vimoir library to interface a java or a python application with Vim through the Netbeans protocol. The client application implements the vimoir {@link vimoir.netbeans.NetbeansEventHandler} interface which maps Vim Netbeans events to methods. The vimoir library takes care of the details of the protocol such as registering buffers with a bufID, managing seqno command sequence numbers, instantiating the client class on each new connection request from Vim and managing the corresponding Netbeans socket connection.

To develop a client, one must implement a class whose constructor signature has a single argument which is an instance of the {@link vimoir.netbeans.NetbeansSocket} interface. The NetbeansSocket object provides the methods to send Netbeans commands and functions. The server instantiates a NetbeansSocket object and the client class on each new incoming connection. The client class is defined by its qualified name which is the value of the vimoir.netbeans.java.client property. The client class must also implement the NetbeansEventHandler interface where are defined the methods used to process Netbeans events.

The syntax of nbkey Vim commands is :nbkey keyName args.... The Netbeans keyAtPos event generated by this Vim command is dispatched to the client class cmd_keyName() method when such a method has been implemented, or to the NetbeansEventHandler default_cmd_processing() method otherwise.

A {@link vimoir.netbeans.NetbeansBuffer} instance is registered to Netbeans by the Netbeans socket on a the first fileOpened event for this buffer. When the client application needs to open a new buffer for which there has been no fileOpened event, a call to the {@link vimoir.netbeans.NetbeansSocket#get_buffer} method returns a NetbeansBuffer object, this object must then be registered to Netbeans with its {@link vimoir.netbeans.NetbeansBuffer#register} method.

The library runs in a single thread of execution. When a time consuming task (such as opening a database for example) must be undertaken on the occurence of a Netbeans event, this task should be run in its own worker thread, so as not to block other I/O operations.
